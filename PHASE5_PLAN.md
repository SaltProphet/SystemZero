# Phase 5 Plan

## Goal
Make System//Zero usable in the field: REST API, background service, enhanced CLI, deployment tooling.

## Scope
1. **REST API** (FastAPI)
   - Capture endpoint: POST /captures
   - Templates endpoint: GET/POST /templates, GET /templates/{id}
   - Logs endpoint: GET /logs, GET /logs/export
   - Status endpoint: GET /status
   - Dashboard data endpoint: GET /dashboard

2. **Enhanced CLI**
   - `run.py server` - start FastAPI server on port 8000
   - `run.py service` - systemd/pm2 integration helpers
   - Help text updates for all commands

3. **Service Tooling**
   - systemd unit file template
   - pm2 config template
   - Background service runner with graceful shutdown
   - Health check endpoint

4. **Tests**
   - API endpoint tests (capture, templates, logs, status)
   - Integration tests (full API workflow)
   - Service runner tests

5. **Documentation**
   - API schema/OpenAPI docs (auto-generated by FastAPI)
   - Deployment guide (systemd/pm2)
   - Service configuration examples

## Implementation Order
1. Create API server module with FastAPI
2. Add endpoints: /captures, /templates, /logs, /status, /dashboard
3. Add request/response models
4. Wire CLI commands: `server`, `service`
5. Create service runner and health checks
6. Add systemd/pm2 config templates
7. Test all endpoints and workflows
8. Update CHANGELOG, ROADMAP, README

## Success Criteria
- All API endpoints return correct responses
- CLI `server` command starts FastAPI on port 8000
- Full integration test passes (capture → API → export)
- Tests: 110+ passing
- Documentation updated
